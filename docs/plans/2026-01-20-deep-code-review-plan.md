## 详细步骤
1. 读取项目结构与入口，划定审查范围与关键路径
2. 深度审查核心模块（命令解析、配置、网络/IO、构建/发布逻辑）
3. 汇总缺陷清单并按严重性排序
4. 修复高优先级问题并补充验证
5. 更新任务与交接文档，提交并推送

## 技术方案
- 通过静态阅读结合搜索定位风险点
- 针对关键路径补充最小验证（单元/集成）
- 对外部接口与数据格式允许破坏性调整以降低复杂性
